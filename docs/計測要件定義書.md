# 計測要件定義書（UNILO ECデモ / GTM・GA4）

本書は、コンサル/マーケが出す「計測の要望（依頼）」と、エンジニアが書いた「実装（どのファイルのどの関数）」を1対1で対比させ、社長が「依頼どおりに実装されているか」を検証するための仕様書です。

- 対象サイト: `ec-gtm-sandbox`（UNILO ECデモ。実購入不可・計測IDはプレースホルダ）
- 実装層: `assets/js/datalayer.js`（計測実装の本体）
- データ層: `assets/js/data.js`（商品マスタの代役 = 本番のサーバー商品DB）
- 状態層: `assets/js/store.js`（カート/会員/注文の代役 = 本番のセッション/DB）
- 共通描画: `assets/js/common.js`（ヘッダー/フッター描画・一覧グリッド描画・全ページ初期化）
- 計測方式: GTM経由でGA4へ送信（gtag.js直貼りはコメント併記のみ。二重計測回避）

検証で照合した実コードに基づいて記述しています（想像値なし）。

---

## 1. 計測アーキテクチャ（サイトのコード → dataLayer → GTM → GA4）

```
┌──────────────────────────────────────────────────────────────────────────┐
│ サイトのコード（ページ / 共通描画）                                          │
│                                                                            │
│  data.js        商品マスタ（item_id/価格/カテゴリ/色/サイズ）                 │
│  store.js       カート・会員・注文（localStorage。本番はサーバーDB）          │
│  common.js      ヘッダー/フッター描画、一覧グリッド描画、全ページ init()       │
│  *.html         各ページが画面イベント時に window.DL.* を呼ぶ                 │
│       │                                                                    │
│       │  window.DL.viewItem(p) / addToCart(...) / purchase(order) ...      │
│       ▼                                                                    │
│  datalayer.js   ← 計測実装の本体。DL.* が値を組み立てて push する            │
└───────────────────────────────┬──────────────────────────────────────────┘
                                 │  window.dataLayer.push({...})
                                 ▼
┌──────────────────────────────────────────────────────────────────────────┐
│ window.dataLayer  （ブラウザ上のイベントキュー / GA4eコマースの器）           │
│   push 例: { ecommerce:null }                                              │
│            { event:'view_item', ecommerce:{ currency:'JPY', value, items}} │
│            { user_id, user_properties:{ login_state, membership_rank } }   │
└───────────────────────────────┬──────────────────────────────────────────┘
                                 │  GTM が dataLayer を読み取り
                                 ▼
┌──────────────────────────────────────────────────────────────────────────┐
│ GTM（Google Tag Manager） ※各HTMLの<head>にスニペット直書き(GTM-XXXXXX)     │
│   トリガー: カスタムイベント名 = event の値（view_item / add_to_cart …）     │
│   変数:     DLV ecommerce / user_id / user_properties.* を取得             │
│   タグ:     GA4 設定タグ（page_view自動）+ GA4 イベントタグ                  │
└───────────────────────────────┬──────────────────────────────────────────┘
                                 │  Measurement Protocol で送信
                                 ▼
┌──────────────────────────────────────────────────────────────────────────┐
│ GA4（Google Analytics 4）                                                  │
│   イベント / eコマース（items[]）/ ユーザープロパティ / user_id            │
└──────────────────────────────────────────────────────────────────────────┘
```

要点:
- 計測の品質は `datalayer.js` が要件どおりに値を積めているかで決まる。
- eコマース系は `DL.ecom()` が毎回 `{ ecommerce: null }` を先に push してリセットしてから本体を push する（前イベントの `items` 混入防止。GA4推奨）。`currency` は全eコマースイベントに自動付与（既定 `JPY`）。

## 2. 計測イベント一覧

| イベント名（GA4） | 発火ページ・タイミング | 呼び出す実装（関数 / ファイル） | 送る主なパラメータ |
|---|---|---|---|
| `page_view` | 全ページ。GTMのGA4設定タグが自動発火（サイト側で手動pushしない。index.html L50に明記） | なし（GTM自動） | page_location / page_title 等（GA4既定） |
| `view_item_list` | 一覧グリッド描画時。`index.html`(おすすめ) / `list.html`(カテゴリ一覧) → `common.js renderProductGrid()` 内 | `DL.viewItemList(listId, listName, products)` / datalayer.js | item_list_id, item_list_name, items[]（index/list情報付き） |
| `select_item` | 一覧の商品カードをクリック（遷移前に同期push）。`renderProductGrid()` が各カードに付与 | `DL.selectItem(listId, listName, p, index)` / datalayer.js | item_list_id, item_list_name, items[1]（index付き） |
| `view_item` | 商品詳細ページ表示時。`product.html` L171 | `DL.viewItem(p)` / datalayer.js | value(=実効価格), items[1] |
| `add_to_cart` | 商品詳細で「カートに入れる」押下。`product.html` L163 | `DL.addToCart(p, quantity, variant)` / datalayer.js | value(=実効価格×数量), items[1]（quantity/variant付き） |
| `remove_from_cart` | カートページで行削除。`cart.html` L120 | `DL.removeFromCart(p, quantity, variant)` / datalayer.js | value(=実効価格×数量), items[1] |
| `view_cart` | カートページ表示時。`cart.html` L107 | `DL.viewCart(items, value)` / datalayer.js | value(=カート合計), items[]（`Store.cartDLItems()`） |
| `begin_checkout` | チェックアウトページ表示時。`checkout.html` L124 | `DL.beginCheckout(items, value, coupon?)` / datalayer.js | value(=カート合計), items[], coupon(任意) |
| `add_shipping_info` | チェックアウトで配送方法を選択。`checkout.html` L132 | `DL.addShippingInfo(items, value, shippingTier)` / datalayer.js | value(=小計+送料), shipping_tier, items[] |
| `add_payment_info` | チェックアウトで支払い方法を選択。`checkout.html` L141 | `DL.addPaymentInfo(items, value, paymentType)` / datalayer.js | value(=小計+送料), payment_type, items[] |
| `purchase` | 注文完了ページ表示時。1注文1回（発火済みフラグでガード）。`complete.html` L52 | `DL.purchase(order)` / datalayer.js | transaction_id, value, tax, shipping, coupon(任意), items[] |
| `login` | ログイン成功時。`login.html` L92 | `DL.login(method)` / datalayer.js | method（既定 `'password'`） |
| `sign_up` | 会員登録成功時。`login.html` L109 | `DL.signUp(method)` / datalayer.js | method（既定 `'password'`） |

補足（eコマースイベント共通）:
- `value` は数値。`view_item`は単価、`add_to_cart`/`remove_from_cart`は単価×数量、`view_cart`/`begin_checkout`はカート合計、`add_shipping_info`/`add_payment_info`は小計+送料、`purchase`は税込合計（送料込み）。
- `currency` は `DL.ecom()` が全eコマースイベントに自動付与する（個別指定不要）。
- `login`/`sign_up` はeコマースではないため `DL.push()` で直接 `{ event, method }` を流す（`ecommerce` を伴わない）。

## 3. eコマース items[] スキーマと取得元

items[] の各要素は `DL.toItem(p, opts)`（datalayer.js L28〜44）が組み立てる。`p` は data.js の商品オブジェクト、`opts` は呼び出し側が渡す `{ quantity, variant, index, listId, listName, price }`。

| キー | 値の取得元 | 備考 |
|---|---|---|
| `item_id` | `p.item_id`（data.js 商品マスタ＝品番） | 必須。GA4のitem_idにそのまま使用 |
| `item_name` | `p.item_name`（data.js） | 必須 |
| `item_brand` | `p.item_brand`（data.js）。無ければ `window.DATA.brand`（='UNILO'） | フォールバックあり |
| `item_category` | `p.item_category`（data.js。大分類 例:メンズ/ウィメンズ） | 日本語名。`categoryKey`(women等)とは別 |
| `item_category2` | `p.item_category2`（data.js。中分類 例:アウター/トップス） | |
| `item_variant` | `opts.variant`。色/サイズを `"色 / サイズ"` 形式で結合 | opts.variant がある時のみ付与。一覧/詳細表示時は無し |
| `price` | `opts.price` 指定があればそれ、無ければ `DATA.effectivePrice(p)`（salePriceがあればセール価格、無ければ定価） | data.js effectivePrice |
| `quantity` | `opts.quantity`。無指定は `1` | |
| `index` | `opts.index`（一覧内の0始まり位置） | `view_item_list`/`select_item` のみ付与 |
| `item_list_id` | `opts.listId` | リスト系イベントのみ付与 |
| `item_list_name` | `opts.listName` | リスト系イベントのみ付与 |
| `currency` | itemには付与しない。イベント直下に `DL.ecom()` が `CURRENCY`(=`DATA.currency`='JPY') を付与 | itemレベルでなくイベントレベル |

variant・list情報の生成元:
- 一覧の `item_list_id` / `item_list_name`: 呼び出し側が指定。`index.html`は `'featured_top'` / `'トップ おすすめ'`、`list.html`は `'category_' + key` / `meta.name + ' 一覧'`。
- カート/チェックアウト/購入の items[]: `store.js cartDLItems()`（L58〜63）が `cartLines()`（カート行×商品マスタ結合）から組み立て、`variant` を `[color, size].filter(Boolean).join(' / ')` で生成して `DL.toItem` に渡す。

## 4. ユーザープロパティ / user_id 仕様

実装: `datalayer.js setUser(user)`（L139〜147）。push内容:

```
{
  user_id: user.logged_in ? user.user_id : undefined,
  user_properties: {
    login_state:     user.logged_in ? 'logged_in' : 'guest',
    membership_rank: user.logged_in ? user.membership_rank : 'none'
  }
}
```

| 項目 | 取得元 | ログイン時 | 未ログイン時 |
|---|---|---|---|
| `user_id` | `store.js login()` がメールハッシュで採番（`'U'+hash`） | `user.user_id` を送信 | `undefined`（送らない） |
| `user_properties.login_state` | `setUser` が判定 | `'logged_in'` | `'guest'` |
| `user_properties.membership_rank` | `store.js rankFromEmail()`（ブロンズ/シルバー/ゴールド。本番は会員DB） | `user.membership_rank` | `'none'` |

setUser の発火箇所:
- 全ページ初期化時: `common.js init()`（L131、`DOMContentLoaded`）が `DL.setUser(Store.getUser())` を呼ぶ。
- ログイン直後: `login.html` L94（`Store.login()`→`DL.login()`→`DL.setUser(u)`）。
- 会員登録直後: `login.html` L111（`Store.login()`→`DL.signUp()`→`DL.setUser(u)`）。
- マイページ: ログアウト時（`mypage.html` L64 のログアウトクリックハンドラ内）。ページ表示時の setUser は `common.js init()` が担当（上記「全ページ初期化時」）。

GTM側: `user_properties.*` を「ユーザープロパティ」、`user_id` を「ユーザー提供データ(user_id)」変数にマッピングする想定。

## 5. 依頼 vs 実装（対比）

社長が「依頼どおりか」を見る対比表。左=マーケの自然文依頼、右=対応する実装（ファイル:関数:引数）。

| # | マーケの依頼文（例） | 対応する実装 | 検証ポイント |
|---|---|---|---|
| 1 | 「商品詳細ページを見たら view_item を送って。値段(value)と商品情報を入れて」 | `product.html` L171 → `datalayer.js viewItem(p)` → `ecom('view_item', { value: DATA.effectivePrice(p), items:[toItem(p,{})] })` | value=実効価格（セール反映）、items 1件、currency自動付与 |
| 2 | 「一覧で表示された商品の並び順(index)も計測したい。どのリストから来たかも分かるように」 | `common.js renderProductGrid()` → `datalayer.js viewItemList(listId, listName, products)` で各itemに `index/item_list_id/item_list_name` 付与 | index は0始まり、listId/listNameは呼び出し側で指定 |
| 3 | 「カート投入は数量と選んだ色・サイズ(variant)も送って」 | `product.html` L163 → `datalayer.js addToCart(p, quantity, variant)`。variantは `"色 / サイズ"` | items に quantity と item_variant が入る。value=単価×数量 |
| 4 | 「購入完了は重複させず、注文番号(transaction_id)・税・送料も送って」 | `complete.html` L52 → `datalayer.js purchase(order)`。重複防止は `store.js isOrderFired/markOrderFired` | transaction_id/value/tax/shipping/items。リロードで再発火しない |
| 5 | 「会員ランクと会員IDをGA4のユーザー属性として全ページで送って。未ログインはguest扱い」 | `common.js init()` L131 → `datalayer.js setUser(user)`。`user_properties.{login_state, membership_rank}` と `user_id` | 未ログインは login_state='guest'/rank='none'/user_id送らない |
| 6 | 「配送方法・支払い方法を選んだタイミングも計測したい」 | `checkout.html` L132/L141 → `addShippingInfo(items, value, shipping_tier)` / `addPaymentInfo(items, value, payment_type)` | shipping_tier / payment_type が入る |

## 6. サーバー注入ポイント（本番での生成箇所）

このデモでは `store.js` / `data.js` が代役だが、本番では以下の値はサーバーが生成し、サーバーサイドテンプレートでHTMLに埋め込む（`datalayer.js` のコメントで「[サーバー注入ポイント]」と明記されている箇所）。

| 注入する値 | デモでの代役 | 本番での生成元・方法 |
|---|---|---|
| `transaction_id` | `store.js genTransactionId()`（時刻+乱数の擬似採番） | 注文DB（注文確定時にサーバーが採番）。サンクスページHTMLに埋め込み（例 `window.__ORDER__`） |
| 購入の `items[]` / `value` / `tax` / `shipping` | `checkout.html` でカートから算出し `store.js saveOrder()`、`complete.html` が `getLastOrder()` | サーバーが注文確定データから生成し、サンクスページに埋め込む |
| `user_id` | `store.js login()` のメールハッシュ採番 | 会員DB。認証後にサーバーが確定 |
| `membership_rank` | `store.js rankFromEmail()` のメールハッシュ割当 | 会員DB（購入履歴等からサーバーが決定） |
| 商品マスタ（item_id/price/category 等） | `data.js` の固定配列 | 商品DB。サーバーが画面ごとに必要分を返す |

検証時の注意:
- `purchase` の `transaction_id` は本番ではサーバー採番のため、サーバー埋め込み変数（例 `window.__ORDER__`）を `DL.purchase()` に渡す形へ差し替える。デモの localStorage 経由ロジックはそのまま本番には載せない。
- 二重計測防止（`isOrderFired`/`markOrderFired`）はデモのlocalStorage実装。本番ではサーバー側のサンクスページ初回表示判定 or GTMトリガー条件で担保する設計に置き換える。
